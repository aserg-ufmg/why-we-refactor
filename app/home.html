Why We Refactor? Confessions of GitHub Contributors - Dataset

This website contains the dataset from the following study:

Danilo Silva, Nikolaos Tsantalis, and Marco Tulio Valente, "Why We Refactor? Confessions of GitHub Contributors", 24th ACM SIGSOFT International Symposium on the Foundations of Software Engineering (FSE'2016), Seattle, WA, USA, November 13-18, 2016. [preprint] [bibtex]

If you are interested to learn more about the process we followed for the creation of this dataset please refer to our paper. The tool we used to detect refactorings is available at GitHub:
https://github.com/tsantalis/RefactoringMiner

Navigable Dataset

  • Studied Projects
  • All Commits with Refactoring (True Positives)
  • Analyzed Commits
  • Thematic Analysis: Themes Before/After Consensus

Dataset in JSON Format

  • All Commits with Refactoring (True Positives)
  • All Thematic Analysis Data

How to contribute

Researchers are welcome to contribute to this dataset either reporting issues or incorporating new data from different projects or commits. Such contributions should be made via GitHub issues or pull requests, by accessing the following repository:
https://github.com/aserg-ufmg/why-we-refactor

Specific instructions on how to incorporate new data are available in the README.
